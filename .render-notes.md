# Render deployment notes

- App uses SQLite.
- `render.yaml` currently mounts a persistent disk at `/opt/render/project/src/data`.
- `DATABASE_PATH` is set to `/opt/render/project/src/data/users.db` for persistence.
- Session storage uses SQLite (`sessions.db`) in the same data directory.
- Startup now validates secrets and admin credentials before boot.
- Keep `ADMIN_USERNAME`, `ADMIN_PASSWORD`, `SESSION_SECRET`, and Paystack keys in Render environment variables, not in code.
